# 聪明小爪股票筛选系统 (Stock System GUI)

一个基于 Tushare 数据源、面向专业投资者的硬核股票筛选与扫描系统。系统采用"原始字段透传"设计哲学,支持全市场 5000+ 股票的实时行情同步、深度财务指标过滤及自定义题材追踪。

---

## 📖 1. 使用指南 (Quick Start)

### 第一步:环境与配置
1.  **获取 Token**:注册 [Tushare](https://tushare.pro/) 并获取您的 API Token。
2.  **配置 Token**:启动程序,进入 `🔧 系统设置` 标签页,填入您的 Token 并保存。
3.  **初始化**:点击菜单栏 `工具 -> 更新数据`,系统将自动同步最新的股票列表。

### 第二步:数据准备 (核心)
1.  **同步行情**:进入 `📋 全部股票` 页面,点击 `📈 更新全市场行情`。系统将按交易日批量补全 2025 年至今的所有历史及当日行情。
2.  **同步财务**:进入 `💰 财务数据` 页面,点击同步。系统将建立全量字段仓库,涵盖 ROE、扣非净利增速、毛利率等 100+ 指标。

### 第三步:硬核筛选
1.  **设置范围**:在 `🔍 扫描筛选` 左侧面板选择"全市场"或指定的"我的版块"。
2.  **配置指标**:在"财务"和"技术"选项卡中设置您的硬性拦截条件(如:ROE > 15%, 扣非净利增速 > 20%, 价格均线多头排列)。
3.  **执行扫描**:点击 `🚀 启动全市场题材扫描`,系统将从 5000+ 标的中秒级锁定符合条件的投资目标。

## 💡 2. 设计思路 (Design Concept)

*   **数据至上 (Data-Centric)**:不依赖第三方加工后的数据,直接对接 Tushare 原始财务报表,确保数据的真实性与穿透力。
*   **硬核透传 (Raw Field Passthrough)**:后端不进行字段含义的二次解释或转换,直接利用 `dt_netprofit_yoy` (扣非净利增速)、`grossprofit_margin` (销售毛利率) 等原始字段进行 SQL 过滤,消除黑盒逻辑。
*   **题材驱动 (Theme-Driven)**:打破官方行业分类的限制,允许用户通过"分组"功能定义自己的"我的版块"(如:AI硬件、人形机器人),并支持在筛选器中针对性扫描。
*   **高性能扫描**:采用 SQLite 关系型数据库存储,结合 Pandas 高性能向量化计算处理技术指标,实现海量数据下的快速反馈。

## ✨ 3. 实现功能 (Features)

*   **全部股票管理**:展示全市场 5000+ 股票,支持 ⭐ 自选标记、序号自动生成、多列点击排序。
*   **高效行情同步**:采用"按交易日批量下载"策略,单次接口调用同步 5000 条记录,支持残缺日期自动重补。
*   **深度财务筛选**:
    *   支持"仅年报"与"最新财报"切换。
    *   全量保存 Tushare 100+ 财务指标,支持无限扩展。
    *   严格数值过滤逻辑,确保结果精准。
*   **技术形态识别**:
    *   均线系统:短/中/长线多头排列识别。
    *   强弱指标:RSI 周期交叉与数值区间过滤。
    *   成交量系统:VMA 放量金叉识别。
*   **策略方案管理**:支持将筛选条件保存为策略模板(JSON 格式),实现一键加载复用。
*   **K线图深度分析**(V2.1 新增):
    *   在「全部股票」「智能筛选」「持仓管理」页面中,双击或右键点击任意股票,即可查看其**过去 300 个交易日的 K 线图**。
    *   图表包含:日本蜡烛图(K线)、成交量柱、RSI(14) 指标(含超买/超卖线 70/30)。
    *   支持 matplotlib 交互工具条(缩放、平移、保存为图片)。
    *   异步加载数据,不阻塞界面操作。
*   **动态止盈止损管理**(V2.0 新增):
    *   **独立策略配置**:每个持仓可设置不同的止盈止损策略
    *   **止损类型**:固定比例止损(默认8%)、移动止损(trailing)、保本止损(breakeven)
    *   **止盈类型**:移动止盈(trailing)、分批止盈(scale)、目标价止盈(target)
    *   **智能参数**:支持调整止损比例、移动模式(strict/loose)、分批阶梯等
    *   **可视化操作**:右键菜单 → "修改止盈止损策略",弹出对话框实时配置
    *   **价格显示**:持仓表格直接显示当前触发价,悬停查看完整策略详情

## 🛠️ 4. 实现路径与方式 (Implementation)

*   **后端架构**:
    *   `Python 3.10+` 作为核心开发语言。
    *   `SQLite3` 作为存储引擎,维护 `daily_trade` (行情)、`financial_indicators` (财务) 和 `stocks_basic` (基础信息) 三大数据库。
    *   `Tushare SDK` 负责与官方接口通讯。
*   **前端界面**:
    *   `Tkinter` + `TTK` 自定义皮肤样式(Clam 主题基础)。
    *   采用三栏式布局与 Notebook 标签页管理。
    *   异步线程处理数据加载(Threading + after 回调),确保界面操作不卡顿。
    *   `stock_chart.py`:基于 `mplfinance` + `matplotlib` 的嵌入式 K 线图组件,支持成交量、RSI 等多面板展示。
*   **逻辑层**:
    *   `FinancialDataManager`:动态 SQL 生成与表结构自适应扩展(Auto-Alter-Table)。
    *   `DailyDataManager`:交易日历对齐与批次下载算法。
    *   `Indicators`:基于 Pandas 的技术指标计算引擎。

## ⚠️ 5. 注意事项 (Precautions)

*   **Tushare 权限**:系统使用的 `fina_indicator_vip` 接口通常需要 2000 以上积分(视 Tushare 官方政策而定),请确保您的 Token 拥有相应权限。
*   **数据完整性**:若同步过程中网络中断导致某日数据不全,系统下一次同步时会自动识别并重补(数据量 < 4000 条即判定为不全)。
*   **首次启动**:建议先清空 `database/` 下的旧数据库文件,让程序自动执行最新的表结构初始化。
*   **排序注意**:在"全部股票"中,"序号"列是根据当前视图动态生成的,不支持排序;点击其他表头可进行逻辑排序。

## 📝 6. 其他说明 (Other Notes)

*   **指标定义**:
    *   `dt_netprofit_yoy`:归属母公司股东的净利润-扣除非经常损益同比增长率(%)。
    *   `grossprofit_margin`:销售毛利率(%)。
*   **未来扩展**:由于采用了全透传设计,后续增加新指标(如:现金流、资产负债比等)仅需修改 UI 配置文件,无需改动核心 SQL 逻辑。

## 🏅 7. 智能评分体系 (Scoring System)

筛选器不再仅仅是硬性拦截,系统内置了一套 100 分制的专家级评分模型,旨在从海量标的中锁定具备"预期差"和"加速度"的极品。

### **评分维度分布 (总分 100)**

1.  **财务质量 (20分)**
    *   **ROE_dt**:>=15% (8分), >=10% (4分)。
    *   **毛利率**:>=35% (8分), >=20% (4分)。
    *   **资产负债率**:<=45% (4分), <=60% (2分)。
2.  **成长性与加速 (25分)**
    *   **营收增速**:>=30% (10分), >=15% (5分)。
    *   **净利增速**:>=30% (10分), >=15% (5分)。
    *   **加速度项**:当"季度营收增速 > 年度均值"时额外加 5 分,识别主升浪前夜。
3.  **技术形态 (20分)**
    *   **趋势**:均线多头排列 (10分)。
    *   **买点**:股价处于 MA20 上方 0%-5% 范围内(刚突破或完美回踩)加 10 分。
4.  **预期差 (20分)**
    *   **启动区识别**:RSI(14) 处于 45-65 的强势起步区(还没被充分炒作)加 15 分。
    *   **低估溢价**:RSI < 55(存在预期提升空间)额外加 5 分。
5.  **资金行为 (15分)**
    *   **放量强度**:基于成交量均线金叉,根据"量比"动态打分。今日量 > 5日均量 1.5倍 (15分), 1.1倍 (8分)。

### **分数释义**
*   **85分以上 (极品)**：基本面无瑕疵、业绩加速且处于放量起步阶段的领头羊。
*   **70-85分 (优质)**：基本面稳健，可能处于高位震荡或回踩阶段，具备中长期持有价值。
*   **低于 60分**：虽符合硬性过滤条件，但可能存在估值过高、动能转弱或业绩放缓等隐忧。

## 🎯 8. 动态止盈止损策略 (Stop-Loss & Profit-Taking)

### 核心设计理念
面向成长股量身定制，遵循"截断亏损，让利润奔跑"原则。
系统自动跟踪持仓最高价，表格直接显示各策略的当前触发价格。

### 止损策略类型

| 类型 | 算法 | 默认参数 | 适用场景 |
|------|------|---------|---------|
| **固定比例止损(fixed)** ⭐默认 | 当前价 < 成本价 × (1 - 比例) | 8% | 通用严格风控，逻辑清晰 |
| **移动止损(trailing)** | 当前价 < 最高价 × (1 - 比例) | 20%，strict模式 | 已盈利持仓，让利润奔跑 |
| **保本止损(breakeven)** | 未激活: 固定比例 / 已激活: 成本价 | 初始8%，激活10% | 先求保本，再求利润 |

### 止盈策略类型

| 类型 | 算法 | 默认参数 | 适用场景 |
|------|------|---------|---------|
| **移动止盈(trailing)** ⭐默认 | 当前价 < 最高价 × (1 - 比例) | 15% | 主升浪中让利润奔跑 |
| **分批止盈(scale)** | 三级阶梯止盈 | +20%/+40%/+60% | 不确定涨幅，分批锁定 |
| **目标价止盈(target)** | 当前价 >= 目标价 | 手动设置 | 有明确估值目标 |

### 移动模式

| 模式 | 说明 |
|------|------|
| **strict(严格)** | 回落到触发价立即执行 |
| **loose(宽松)** | 触发价额外下移2%，减少震荡被震出 |

### 分批止盈详解

```
成本价: 100元

第1批 +20% (120元) → 卖33%
  锁定部分利润，剩余仓位继续持有

第2批 +40% (140元) → 再卖33%
  大部分利润已锁定，剩余仓位零成本

第3批 +60% (160元) → 启动移动止盈
  从最高点回落8%清仓，让最后利润奔跑
```

### 使用方法

1. **查看价格**：持仓表格"止损价""止盈价"列直接显示当前触发价
2. **查看详情**：鼠标悬停在价格上，弹出 tooltip 显示完整策略参数
3. **设置策略**：右键点击股票 → "修改止盈止损策略"
4. **调整参数**：根据股票特性和个人偏好调整比例
5. **实时监控**：系统自动跟踪最高价，触发条件时返回操作建议

## 🤖 8. 自动化与 API 接口 (Automation)

系统支持通过 `scripts/auto_tasks.py` 进行无界面自动化操作,方便 AI 助手或定时任务调用。

*   **批量添加自选**:`python scripts/auto_tasks.py --add-watchlist "代码1,代码2" --group "分组名"`
*   **批量生成报告**:`python scripts/auto_tasks.py --gen-reports positions` (为持仓股生成体检报告)
*   **一键 AI 审计**:`python scripts/auto_tasks.py --ai-audit`
*   **自动同步行情**:`python scripts/auto_tasks.py --sync-market`

详细指令请参考 `README_AUTO.md`。
